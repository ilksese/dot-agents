# seamaid-code 项目根目录通用化设计

## 背景

当前 `agents/seamaid-code.md` 将公司项目根目录写死为 `$HOME/Projects`，跨设备/用户时需要手动修改。目标：让 agent 自动适配不同环境。

## 方案

两级优先级：

1. **环境变量 `SEAMAID_PROJECT_ROOTS`** — 如果设置了，用冒号分隔多个项目根目录。例如 `~/Projects:/Volumes/work/frontend`。
2. **当前项目父级目录** — 如果未设置环境变量，自动推断当前工作仓库的父目录作为项目根目录。

## 具体改动

### 路径约定（新增）

- 新增 `SEAMAID_PROJECT_ROOTS` 环境变量约定，说明冒号分隔多目录
- 新增回退规则：未设置时取当前项目父级目录

### 项目范围

- 删除"公司的前端项目位于 `$HOME/Projects`"的硬编码
- 改为"公司前端项目位于项目根目录集合中"

### 项目获取规则

- 两步：确定根目录集合 → 扫描每个根目录
- 推荐命令改为动态构建

### 已知项目线索

- 保持为纯线索，不绑定到特定根目录

## 未涉及

- 公司项目发现逻辑（`next-*-web` 命名规则）不变
- agent 其他职责不变
- 不增加新依赖
